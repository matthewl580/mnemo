# Brand and Trademark Policy

This policy covers the Mnemo name, logo, icons, visual identity, website branding, and other project marks.

## Ownership

The Mnemo name, logo, visual identity, and official branding are controlled by **@torstfugl** for the official Mnemo project under the **@onemnemo** GitHub organization.

The project license for the source code does not grant permission to use Mnemo branding except as described here.

## What you may do

You may:

- refer to the official project as Mnemo
- link to the official repository or website
- accurately state that your work is based on Mnemo
- accurately state that your work is compatible with Mnemo
- accurately state that your project is a fork of Mnemo

## What you may not do without permission

You may not:

- present your fork, build, website, service, or product as the official Mnemo project
- use the Mnemo name or logo in a way that suggests endorsement
- use the Mnemo logo as the main branding for a fork or redistributed build
- register domains, social accounts, app store listings, packages, or services that imply they are official Mnemo channels
- sell or provide services using Mnemo branding in a way that confuses users about who operates the service

## Forks and redistributed builds

If you distribute a modified version of Mnemo, please use a different name and visual identity unless you have written permission.

You may still say something like:

> This project is a fork of Mnemo.

or:

> Based on Mnemo, an open-source study application.

The important part is that users must be able to tell that your version is not the official Mnemo project.
